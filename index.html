    What is MultiMail?

    MultiMail is an offline mail packet reader for Unix and other systems. (This version also compiles under MSDOS, with DJGPP or Turbo C++; OS/2, with EMX; Win32, with RSXNT or Borland C++; BeOS; and AtheOS.) It currently supports the Blue Wave, QWK, OMEN and SOUP formats. It has a full screen, color user interface, built with the curses library.

    MultiMail is free, \"open source\" software, distributed under the GNU General Public License (q.v.).

    Please direct all correspondence about MultiMail to William McBrine (${EMAIL}).

    Features

    • auto-decompression of packets with external compress program
    • user-friendly menus to select packet, area, letter, etc.
    • save whole area or one letter in a text file
    • enter mail in any area (using external editor)
    • insert tagline from a tagline file
    • reply mail with quote
    • write netmail
    • netmail addressbook

    Requirements for compiling

    • Unix-compatible (POSIX) API, or Borland/Turbo C++
    • curses -- tested with ncurses 1.9.3+, Solaris curses, and PDCurses
    • GNU make may be required on some systems

    Binaries

    Precompiled executables of v${VER} for various systems. If you can submit binaries for other platforms, please email me.

    Unix

    I don't actually recommend that you use these. They're statically linked with ncurses, but dynamically linked otherwise; so you might do better compiling your own. Note that if you want the documentation, you'll also need the source archive; these are only the binaries.

    To use any of these, simply gunzip (NOT untar -- these are not tar files), make executable (e.g., \"chmod 700 mm-linux-i386elf\"), and run.

    • mm-linux-i386elf.gz (${KsizeLinux}k) - Linux/i386 ELF, compiled on a modified Slackware 3.0. Needs libc5.
    • mm-netbsd-m68k.gz (${KsizeNetBSD}k) - NetBSD/m68k, compiled on 1.3.
    • mm-qnx-i386.gz (${KsizeQNX}k) - QNX/i386, compiled on RTP 6.0.
    • mm-solaris-i386.gz (${KsizeSolaris}k) - Solaris/i386, compiled on 8.
    • From the SourceForge compile farm -- most of these are dynamically linked with ncurses:
      • mm-darwin-ppc.gz (${KsizeDarwin}k) - Darwin/PPC, aka Mac OS X. Yes, finally a Mac port! (Sort of.)
      • mm-linux-alpha.gz (${KsizeAlpha}k) - Linux/Alpha, compiled on Debian 3.0.
      • mm-linux-arm.gz (${KsizeARM}k) - Linux/ARM, compiled on Debian 3.0.
      • mm-linux-sparc.gz (${KsizeSparc}k) - Linux/Sparc, compiled on Debian 3.0.
      • mm-solaris-sparc.gz (${KsizeSolaris2}k) - Solaris/Sparc, compiled on 8. This one is statically linked with ncurses.

    BeOS

    • mm-beos-r5-intel.gz (${KsizeBeOS}k) - For R5 Intel. This is a straight Unix-like compile, taking advantage of BeOS' POSIX environment, with no mods for BeOS. (I almost put it in the section above.) This is only the binary; no docs. As with the Unix binaries, gunzip it and make it executable.

    MS-DOS

    • mmdos${SHORTVER}.zip (${KsizeDOS}k) - 32-bit version. This also includes documentation. Needs at least a 386 to run. Also works under Windows 9x, etc.
    • mmxt${SHORTVER}.zip (${KsizeXT}k) - 16-bit version. This also includes documentation.

    OS/2

    • mmos2${SHORTVER}.zip (${KsizeOS2}k) - This also includes documentation. Needs at least OS/2 2.0 and the EMX runtime library.

    Win32

    • mmwin${SHORTVER}.zip (${KsizeBCC}k) - Borland C++ version. This also includes documentation. Needs Windows 9x, ME, NT, 2000 or XP (recommended for NT, 2000 or XP). Note that this is a console (text-mode) application.

    AtheOS

    • mmail-${VER}.bin.1.tgz (${KsizeAtheOS}k) - For AtheOS, in its standard package format. (Only the binary and man page are included.) You'll also want to install unzip, which is available in the \"Optional packages\" section of the AtheOS site, and zip, which I'm making available as a package here (65k). (You'll also need an editor -- the standard AtheOS mc package works nicely, and provides a link to \"vi\", which is MultiMail's default.) As with the BeOS port, this is a straight POSIX compile.

    Additional port status

    As an alternative to the Win32 versions, the 32-bit MSDOS version supports long file names when run under Windows 9x and ME (but not NT). In fact, on those platforms, I recommend it over the native Win32 port. But for NT, 2000 or XP, you should use the native Win32 version.

    Other ports that might be interesting:

    • Amiga OS
    • Atari MiNT

    Unzip, with unshrinking

    Archives produced by PKZIP 1.1 (reportedly still used on some BBSes!) can contain files compressed with the \"shrinking\" method, which is based on the now-infamous LZW algorithm. Because of the patent problems associated with LZW, the binaries distributed by the InfoZip group have unshrinking disabled, but it's still included in the source code. For your convenience, here are binaries of Unzip 5.41 with unshrinking included, for OS/2 and Win32:

    • OS/2 (42k) - Needs at least OS/2 2.0 and the EMX runtime library.
    • Win32 (87k) - Needs Windows 9x, ME, NT, 2000 or XP. Note that this is a console (text-mode) application.
